PWA

A beginner's workshop

We call a website a "Progressive Web App" if it behaves like an app - meaning the design is responsive, it's downloadable to your phone (iPhones too, since last year, whoop whoop) and has offline capabilities.

The app should work across different browsers and ensure backwards-compatibility for older browsers (hence "progressive").

Every website can be turned into a PWA with just some little adjustments, doesn't matter if it's written in React or Angular, or just plain old Javascript/CSS/HTML. Here we'll stick with the latter.

By adding a Service Worker file we'll optimize the caching of the browser and control the offline capabilities.

With a Manifest file we define the metadata necessary to add the app to our phone`s home screen.

Benefits...

PWAs give the owner the best of both worlds: While providing a "native" feeling for the app users, they are accessible from all devices with minimum development cost and effort.

PWA development is also far faster and easier than native development.

... and weaknesses

You cannot find your PWA on one of the app stores.

The access to some of the hardware features of the device is limited, and not all devices support the same range of PWA features (yet).

iOS so far does not support Web Notifications; Background Sync and a couple of the Manifest file's attributes.

Checklist

A progressive webpage must fulfill all the following requirements:

  • Served via HTTPS
  • Pages are responsive on mobile devices
  • The app has a serviceworker file and all app URLs load while offline
  • The app has a Web App Manifest file to define the metadata
  • Fast first loading time
  • Site works cross-browser

You can check your PWA with Lighthouse in Chrome Developer Tools.
